# AI導入支援の参照モデル

## 支援フェーズ

| Phase | 目的 | 期間目安 | 主な判定 |
|---|---|---|
| Discovery | 課題・意思決定構造の把握 | 1-2回 | stakeholder_alignment |
| Design | 運用設計・責任分界の固定 | 2-4週 | operating_clarity |
| Pilot | 小さな現場単位で検証 | 4-8週 | field_readiness |
| Rollout | 定着・横展開 | 8-12週 | rollout_risk |

## 支援カテゴリ

1. **合意形成**：意思決定者向け資料、社内説明台本、費用対効果の整理
2. **運用設計**：責任分界、例外処理、更新手順、利用ルール
3. **現場定着**：小さな成功例、チャンピオン設計、週次レビュー
4. **展開管理**：90日ロードマップ、指標固定、ステークホルダー調整

## GO / HOLD / STOP 判定基準

- **GO**：合意・運用・現場準備がそろい、展開リスクが低い
- **HOLD**：運用ルールが曖昧、または意思決定者の納得度が不足
- **STOP**：責任者不在、例外処理未定義、現場負荷が読めない
